Integration Recommendations
Decisions needed

🔐 Auth — Social Login
✓ Use: python-social-auth + Google OAuth2
Free, self-hosted, no monthly fee. Add google-oauth2 backend to auth router. One redirect URL on Google Cloud Console. Stores google_id alongside email/password — users can log in either way.
✗ Skip Auth0: $23/month minimum, unnecessary for 300 local users.
💳 Payments — Primary
✓ Use: Tap Payments (tap.company)
Oman-based, supports OMR, accepts Mada, KNET, Visa/MC local cards. Webhook stub already in payments/router.py. Register at tap.company. Approval ~1 week.
✗ Not Stripe as primary: USD-first, currency conversion fees hurt OMR feel.
💳 Payments — Secondary
✓ Use: Stripe (for expats + Apple Pay)
Large expat community in Oman uses international cards. Stripe powers Apple Pay + Google Pay. Already stubbed. Activate both — Tap for locals, Stripe for expats.
✗ Don't skip COD: "Meet in Person" is how most Oman MTG trades happen. Add first.
📧 Email
✓ Use: Resend.com
3,000 emails/month free. Simple REST API. Professional transactional email from your domain. Order confirmations, price alerts, account verification.
✗ Skip SendGrid / Mailgun: paid tiers needed sooner, more complex setup.
📱 WhatsApp Scanner
✓ Use: Twilio WhatsApp Business API
Scanner endpoint already exists (POST /api/scanner/identify). Twilio receives image, forwards to your API, sends price response back. Sandbox free for testing. ~$0.005/message production.
Alternative backend: Scryfall visual search already does card image matching.
🗺️ Deck Import
✓ Use: Plain text parser (no external API)
Moxfield, Manabox, MTGO, Arena all export "4x Lightning Bolt (M10)" format. Parse with regex, look up by name in cards table. Free, instant, covers every major platform.
No need for Moxfield API — the text export format is standard.
🔄 Price Sync
✓ Use: Scryfall bulk-data weekly cron
Scryfall free daily bulk JSON (~280MB compressed). Run weekly: download → UPDATE cards SET prices=... No API key needed. Already have the Scryfall client module.
Just add a cron / Windows scheduled task to trigger the sync script.
🏗️ Production DB
✓ Use: PostgreSQL 16
config.py already has DATABASE_URL pointing to PostgreSQL. SQLite stays for local dev. PG gives concurrent writes, GIN indexes for faster JSON search, pg_trgm for fuzzy name matching.
SQLite will lock under concurrent writes in production.
